## Bug 2 — Triage notes display the wrong content on mobile

### Context

The web app stores queue metadata in `patient_queue.triage_notes` as a JSON
string with this shape:

```json
{
  "queue_number": "ECG-001",
  "vitals": {
    "blood_pressure": null,
    "weight_kg": null,
    "temperature_c": null
  },
  "notes": ""
}
```

The receptionist's actual typed note ("Testing" in the bug report screenshot)
goes into the `notes` field of that JSON.

The mobile app is currently displaying the raw JSON string. It should be
**parsing the JSON and displaying only the `notes` field** as the triage note.

### Files affected (likely list — confirm by reading the actual files)

- `lib/features/staff/presentation/widgets/queue_entry_card.dart` (where the
  triage notes are rendered inline)
- A patient queue detail modal/screen (the second mobile screenshot in the
  bug report shows the same JSON, so this rendering happens in at least two
  places)

### Changes to make

1. **Create a small helper** `lib/core/utils/triage_notes_formatter.dart`
   with one function:

   ```dart
   /// Extracts the human-readable notes string from the triage_notes field.
   ///
   /// The web portal stores triage_notes as a JSON object containing
   /// queue_number, vitals, and a notes field. The mobile app only needs
   /// to display the notes field to staff.
   ///
   /// If the input is not valid JSON (e.g., a legacy entry stored as plain
   /// text), the original string is returned as-is.
   ///
   /// If the input is valid JSON but the notes field is empty or null,
   /// returns null (caller should suppress display).
   String? extractTriageNotes(String? rawTriageNotes) {
     if (rawTriageNotes == null || rawTriageNotes.trim().isEmpty) {
       return null;
     }
     try {
       final parsed = jsonDecode(rawTriageNotes);
       if (parsed is Map<String, dynamic>) {
         final notes = parsed['notes'];
         if (notes is String && notes.trim().isNotEmpty) {
           return notes.trim();
         }
         return null;
       }
       // JSON but not the expected shape — fall through to raw display
       return rawTriageNotes;
     } catch (_) {
       // Not JSON — likely a legacy plain-text entry. Display as-is.
       return rawTriageNotes;
     }
   }
   ```

2. **Update every place that displays `triage_notes`** to use this helper.
   The pattern:

   ```dart
   // Before:
   Text(queueEntry.triageNotes ?? '')

   // After:
   Builder(builder: (context) {
     final notes = extractTriageNotes(queueEntry.triageNotes);
     if (notes == null) return const SizedBox.shrink();
     return Text(notes, style: ...);
   })
   ```

   Or hide the entire "Triage Notes:" label/row when notes is null.

3. **Apply this in all three roles' views** — receptionist, department staff,
   medical specialist (the medical specialist's patient detail likely also
   shows queue history with triage notes).

### Tests

Add `test/phase9_triage_notes_formatter_test.dart` with these unit test
cases for the helper:

| Input | Expected output | Reason |
|---|---|---|
| `null` | `null` | Empty input |
| `""` | `null` | Empty input |
| `'{"notes":"Testing","queue_number":"ECG-001"}'` | `"Testing"` | Normal case |
| `'{"notes":"","queue_number":"ECG-001"}'` | `null` | Empty notes |
| `'{"notes":null,"queue_number":"ECG-001"}'` | `null` | Null notes |
| `'{"queue_number":"ECG-001"}'` | `null` | Missing notes field |
| `'Plain text note'` | `"Plain text note"` | Legacy plain-text fallback |
| `'  {"notes":"  Testing  "}'` | `"Testing"` | Whitespace trim |
| `'{invalid json'` | `'{invalid json'` | Malformed JSON falls through |

### Verification at completion

Paste in the walkthrough:

```bash

# Expected: all 9 cases pass

flutter test
# Expected: All tests passed!
```
